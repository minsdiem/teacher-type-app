# 유지보수 및 업데이트 가이드 (Maintenance Guide)

이 문서는 로컬에서 코드를 수정한 후, 배포된 웹사이트에 반영하는 방법을 설명합니다.

## 전체 흐름
1. **로컬 수정**: VS Code에서 코드 수정 및 저장.
2. **GitHub 업로드**: 변경 사항을 GitHub에 업로드 (`git push`).
3. **자동 배포**: Vercel이 GitHub의 변경을 감지하고 자동으로 재배포.

## 상세 단계

### 1. 코드 수정 (Local Edit)
VS Code에서 원하는 파일을 수정하고 저장합니다.

### 2. 터미널 명령어 입력 (Git Push)
수정이 끝났으면, VS Code 하단의 터미널(Terminal)에 다음 명령어 3개를 순서대로 입력합니다.

```bash
# 1. 변경된 모든 파일을 스테이지에 올림
git add .

# 2. 변경 사항을 설명하는 메시지와 함께 커밋 (따옴표 안에 설명 입력)
git commit -m "수정 내용 설명 (예: 버튼 색상 변경)"

# 3. GitHub로 업로드 (이 명령어가 실행되면 Vercel 배포가 시작됨)
git push
```

### 3. 배포 확인 (Verify)
- `git push`가 성공하면, Vercel 대시보드에서 자동으로 **Building**이 시작됩니다.
- 약 1~2분 뒤 배포가 완료되면, 기존 웹사이트 주소로 접속했을 때 변경 사항이 적용된 것을 볼 수 있습니다.

## 주의 사항
- `git push` 시 에러가 발생하면, 보통 원격 저장소와 내용이 달라서 그렇습니다. 그럴 땐 `git pull`을 먼저 해보세요.
- Vercel 배포 실패 시 Vercel 대시보드의 **Logs**를 확인하면 원인을 알 수 있습니다.
